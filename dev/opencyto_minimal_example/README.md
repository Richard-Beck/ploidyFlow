# OpenCyto preprocessing prototype

This folder contains the current OpenCyto-based preprocessing prototype for the ploidyFlow datasets. The gating workflow and agreed peak detection developed here are intended to become the main, and initially only, preprocessing path used by this repository.

The current gating templates and peak detection approach work well for the datasets available now. They may need extension or dataset-specific modification as new data becomes available, but the overall framework should remain intact: dataset configuration, reproducible OpenCyto gating, debug/QC summaries, agreed tumor peak detection, and downstream inspection plots.

## Migration note

The previous preprocessing steps in `workflow/data_prep.Rmd` should eventually be deprecated. Downstream analyses that currently depend on those Rmd-derived outputs will need to be updated to consume outputs generated by this OpenCyto workflow instead.

Until that migration is complete, this folder should be treated as the candidate source of truth for preprocessing logic rather than as a disposable exploratory branch.

## Contents

- `config/`: dataset-specific OpenCyto configuration, channel maps, gating templates, and job definitions.
- `R/`: helper code for channel mapping, transformations, template validation, custom gates, and configured plot rendering.
- `run_minimal_opencyto_jobs.R`: runs the configured multi-dataset OpenCyto workflow.
- `run_minimal_opencyto_template.R`: runs a single configured OpenCyto template, mainly useful for focused debugging.
- `dev_tumor_gate_lab.R`: development script used to inspect and tune the tumor gate.
- `dev_tumor_singlet_gate_lab.R`: development script used to inspect and tune tumor singlet gating.
- `dev_peak_detection_lab.R`: agreed DNA-A peak detection workflow.
- `plot_agreed_tumor_peaks_by_sample.R`: inspection plots for agreed tumor peak positions and adjacent peak ratios, with QC-flagged samples highlighted.

Generated folders are intentionally not tracked:

- `output/`
- `plots/`
- `PeacoQC_results/`

These are reproducible from the source code and configs.

## Current workflow

Run from the repository root:

```sh
Rscript dev/opencyto_minimal_example/run_minimal_opencyto_jobs.R .
Rscript dev/opencyto_minimal_example/dev_peak_detection_lab.R .
Rscript dev/opencyto_minimal_example/plot_agreed_tumor_peaks_by_sample.R .
```

The first command gates the configured datasets and writes gated flow sets, debug tables, and configured gating figures under `output/`.

The second command performs the current agreed peak detection and writes peak call tables plus per-sample diagnostic figures under `output/peak_detection_figures/`.

The third command writes the current high-level inspection plots under `output/peak_consistency/`:

- `agreed_tumor_peaks_by_sample.png`
- `tumor_peak_ratio_by_sample.png`

It also writes `output/peak_detection_figures/agreed_tumor_peaks_by_sample.csv`, which includes inferred cell line labels, ordered agreed tumor peak labels, and QC flags used by the plots.

## Dataset notes

Current cell-line inference rules used by `plot_agreed_tumor_peaks_by_sample.R` are intentionally dataset-specific:

- `Anoxia_FlowCytometry`: `Sample_CEN.fcs` is excluded from tumor peak inspection. `Sample_SUM159_4N_Control.fcs` is treated as `SUM159_4N`; all other tumor samples are treated as `SUM159_2N`.
- `Hypoxia_SUM159`: filenames containing `_2N_` are treated as `SUM159_2N`; filenames containing `_4N_` are treated as `SUM159_4N`.
- `Polyploidization_EthanolFixation`: the first filename token before `_` is used as the cell-line identifier, except `DKMG` samples are mapped to `SUM159_2N`.

Agreed tumor peaks are ordered within each sample by raw DNA-A and labelled `Tumor 1`, `Tumor 2`, and `Tumor 3` for plotting. These labels are ordered peak identifiers only; they should not be interpreted or renamed as biological `2N`, `4N`, or `8N` states without additional validation.

## QC flags

The current inspection plots flag samples with low retained event counts after gating. The current threshold is:

```text
cells_keep_n < 7000
```

At the time this README was written, the flagged samples were:

- `HCC1937_DCB_A3_018.fcs`
- `Jurkat_001.fcs`
- `KALS1_DCB_A3_008.fcs`
- `MDAMB231_DCB_A4_014.fcs`

These flags are QC annotations only. They are not based on whether the peak positions or peak ratios look unusual.

## Git hygiene

Track source code, configs, and documentation. Do not track generated gated flow sets, debug outputs, or plots. The relevant generated directories are ignored in the repository `.gitignore`.
